3. CALIBRATION STEPS
────────────────────

Step 1: Calibrate HuskyLens
- Power on robot with HuskyLens connected
- In HuskyLens menu: Select ball detection algorithm
- Point at real balls on field
- Adjust detection threshold until balls are highlighted
- Save settings

Step 2: Mount AprilTags
- Mount red goal AprilTag on red side goal
- Mount blue goal AprilTag on blue side goal
- Ensure clear line-of-sight from robot
- Approximately 3-4 feet mounting height

Step 3: Verify Turret Encoder
- Manually rotate turret to known position
- Note encoder reading in telemetry
- Rotate 360° and verify reading returns to same value
- If not, adjust ticksPerTurretRotation in TurretConfig

Step 4: Test Pedro Pathing
- Run existing autonomous with manual paths
- Verify robot follows paths accurately
- If drift detected, recalibrate odometry


4. CONFIGURATION TUNING
───────────────────────

File: BallDetectionConfig
- minBlockSize: Start at 20, increase if detecting noise
- maxBallDistance: 500 pixels from center (frame width 320)
- Result: Only realistic ball-sized objects detected

File: AutoSystemConfig
- ballPickupDistance: 8 inches (adjust for intake range)
- turretPositionTolerance: 50 ticks (±50 = ~3.7°)
- turretVelocityTolerance: 100 RPM (flywheel tolerance)
- Timeouts: Start with defaults, increase if timeouts occur

File: Turret.java
- targetVelocity: 1300 (close range), 1600 (far range)
- Adjust based on your goal distance


5. FIRST RUN - AUTONOMOUS MODE
───────────────────────────────

Pre-Match Setup:
1. Place robot on field
2. Select "Auto Ball Pickup & Scoring" OpMode
3. Press INIT
   → System initializes all components
   → Wait for "READY TO START" message
4. Press START
   → Robot waits for command

During Match:
1. Press gamepad1.Y
   → Auto mode enabled (haptic feedback)
   → Robot begins ball scanning
2. Robot will:
   a) Find nearest ball
   b) Navigate to ball
   c) Pick up ball
   d) Navigate to goal
   e) Aim turret
   f) Fire 3 shots
   g) Return to scanning
3. Press gamepad1.X to cancel anytime
4. Press gamepad1.Back for emergency stop (all systems halt)


6. HYBRID MODE - TELEOP WITH AUTO ASSIST
──────────────────────────────────────────

Pre-Match Setup:
1. Select "TeleOp with Auto Assist" OpMode
2. Press INIT → System initializes
3. Press START → Ready for driver input

During Match:
1. Normal TeleOp controls work (drive, manual turret, manual intake)
2. Press gamepad1.Y to enable auto assist
   → Turret/intake controlled by system
   → Drive still under manual control
3. Press gamepad1.Y again to return to full manual
4. Emergency controls always available


7. UNDERSTANDING THE STATE MACHINE
───────────────────────────────────

5 Main States:

┌─────────┐
│ MANUAL  │ ← Driver control (start here)
└────┬────┘
     │ Y button pressed
     ▼
┌─────────┐
│  SCAN   │ ← Looking for balls
└────┬────┘
     │ Ball found
     ▼
┌──────────────────┐
│ NAVIGATE_TO_BALL │ ← Following path
└────┬─────────────┘
     │ Ball reached
     ▼
┌────────┐
│ PICKUP │ ← Intake spinning
└────┬───┘
     │ Intake complete
     ▼
┌──────────────────┐
│ NAVIGATE_TO_GOAL │ ← Moving to goal
└────┬─────────────┘
     │ Goal reached
     ▼
┌─────────────────────┐
│ ALIGN_TURRET/SHOOT  │ ← Aiming & firing
└────┬────────────────┘
     │ Shots complete
     ▼
┌────────┐
│ VERIFY │ ← Brief delay
└────┬───┘
     │ Loop back
     └────→ SCAN


8. TELEMETRY READING
────────────────────

Key Information Displayed:
- "Auto Mode": ON/OFF status
- "Current State": Which state machine state
- "Balls Detected": Count of visible balls
- "Tags Visible": Which AprilTags currently seen (R/B)
- "Turret Angle Error": Current accuracy
- "Flywheel Velocity Error": RPM accuracy

If something wrong:
- Check "Current State" (should progress through states)
- Check "Balls Detected" (> 0 required to start)
- Check "Tags Visible" (R or B should appear regularly)
- Check errors if state hangs (suggests timeout)


9. TROUBLESHOOTING QUICK FIXES
──────────────────────────────

Problem: "Balls Detected" shows 0
Fix: 
  - Check HuskyLens connected
  - Verify detection algorithm selected in HuskyLens menu
  - Increase minBlockSize in BallDetectionConfig

Problem: State stuck in SCAN
Fix:
  - Verify ball is in frame
  - Check HuskyLens is looking at correct direction
  - Increase ballDetectionTimeout

Problem: State stuck in NAVIGATE_TO_BALL
Fix:
  - Check robot is moving (look at drive)
  - Verify odometry calibration
  - Increase pathFollowingTimeout

Problem: State stuck in ALIGN_TURRET
Fix:
  - Check turret is moving manually first
  - Verify turret encoder connection
  - Check turretPIDCoef values not too low

Problem: No haptic feedback
Fix:
  - Check enableHapticFeedback = true
  - Test gamepad separately
  - Verify gamepad firmware updated

12. PERFORMANCE OPTIMIZATION
────────────────────────────

If experiencing lag:
1. Reduce vision resolution
2. Lower camera streaming frequency
3. Disable unused debug telemetry
4. Check for memory leaks in telemetry

If autonomous moving slowly:
1. Increase pathFollowingTolerance (looser endpoint)
2. Increase motor speeds in Path configuration
3. Check robot battery voltage
4. Verify motor PID tuning
